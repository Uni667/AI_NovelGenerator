# 相似度防照抄守卫指南 (Similarity Guard)

## 1. 为什么需要防照抄？
大语言模型在注入了极强的参考资料上下文（特别是特定作者的风格圣经、专有名词后），有很高的概率会产生**惰性**，直接原封不动地照搬长篇大论的原著段落（融梗/照抄）。这在商业写作上是不可接受的，具有严重的版权风险。

## 2. 守卫触发机制
系统在章节生成的**最后阶段**（Draft/Revise 之后），会拦截生成的完整章节，并使用多种指纹算法将其与绑定的本地书库进行对比：
1. **N-gram 重叠率**：检查四字连串（n=4）的命中率，如果超出阈值（如5%），则判定过高。
2. **长句连读重叠**：如果在生成的文本中发现了超过一定字数（如 50 字）的完全一致片段，立刻判定为抄袭。
3. **专有名词碰撞**：检查原创名词之外的、属于参考书独占的名词的出现频率。

## 3. 自动打碎重写
一旦防照抄守卫报警（`needs_rewrite=True`）：
- 系统会立刻拦下此草稿，不再发往下一个管道。
- 提取报警特征（比如“这段50字的文字照抄了原文”）。
- 使用一个极具对抗性的指令 Prompt（如：“请你完全抛弃原文的句子结构，用截然不同的语气和节奏彻底打碎重塑这段情节，坚决杜绝原话！”）调用 LLM 对该章进行重做。
- 重写后**再次**进入查重校验。
- 最多允许 2 次自动重写循环，以在时间消耗与绝对安全之间取得平衡。

## 4. 查重报告查看
即使通过了查重（或重写后通过），系统也会在项目的工作区生成一份 `chapter_X_similarity_report.json`。
你可以随时在前端的【本地参考库】面板中获取当前章节的查重分数、N-gram 率，做到心中有数。该报告只会展示数值和命中的偏移量，绝不保留原文本，确保了极度的安全性。

## 5. 安全兜底
哪怕你忘记开启防照抄，系统核心代码也不会将任何参考书的原著原段落保存到主干数据库或者项目日志中。所有操作都在内存和受限的 `manifest` 沙盒内完成。
