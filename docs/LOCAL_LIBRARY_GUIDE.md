# 本地参考书库使用指南

## 1. 简介
本地参考书库（Local Library）功能允许创作者将本地的 TXT/MD 小说文件或资料文件夹一次性挂载到系统中。系统可以扫描多层目录结构，提取小说文本，并在后续步骤中执行深度的内容吸收（如章节摘要、风格圣经、场景拆解等），进而使得后续生成的章节在风格和设定上与参考书籍保持高度统一。

## 2. 挂载本地库
由于浏览器的安全限制，直接读取大型本地文件夹可能受到阻碍，因此我们通过 **后端配置** 来打通本地系统：
1. 打开系统左侧导航栏的 **本地参考书库** 面板。
2. 在“路径配置”中，填写小说源文件夹的绝对路径（例如 `D:/MyBooks`）。
3. 填写“精华存储路径”，这是系统提取并消化完本地库后产生的 `manifest.json` 与各维度的摘要产物的存放位置。
4. 确保在后端 `.env` 中开启了 `ALLOW_LOCAL_FILE_ACCESS=true`。
5. 点击**配置并测试**，测试通过后可直接进行全量扫描。

## 3. 扫描过程与状态
点击扫描后，系统将以后台任务的形式遍历你的书库文件夹。
支持：
* 多种编码自动探测（UTF-8、GBK、Big5）
* 自动提取文件大小与修改时间
* 检测增量变更（不重复扫描已解析过且未修改的书）
* 跳过超过指定上限（如 10MB）的过大文件，防止内存溢出

## 4. 章节识别边界修正
挂载书籍后，你可以点击某本书查看解析结果。系统内置了强大的正则表达式解析器，能自动识别“第x章”、“卷x”、“番外”等结构。
如果你发现章节识别存在偏差，可以通过面板的“边界审核”进行人工干预和重新定义边界。

## 5. 从书库到项目绑定
一旦书籍进入书库并被“吸收”，你就可以前往具体项目的【本地参考库】标签页，将特定的书籍绑定进该项目。
绑定后，你可以针对不同的书调节“权重”，并决定是否开启【防照抄守卫】功能。
