# 备份与回滚指南

在长篇小说创作中，设定就是作者的心血。为了防止误操作、误合并或大模型幻觉破坏设定，本项目搭载了完整的状态审计与备份系统。

## 1. 自动备份触发机制

你不需要时刻点击“保存备份”。系统会在以下高风险节点自动触发无感知的备份：
- 你在 **StateTab（状态页）** 中手动修改了任何一条设定。
- 你在页面上点击了 **“合并 (Merge)”** 一个大模型生成的 State Patch。
- 你点击 **“应用 (Apply)”** 了一个大模型生成的大纲演化建议 Outline Diff。

备份文件会存储在项目的 `memory/backups/` 目录下，并以时间戳命名。

## 2. 审计日志

为了让你清楚“某天某时某刻设定的修改是谁做的？为什么做？”，系统会在每次写入记忆文件时，向 `memory/audit_logs/state_audit.jsonl` 中写入一条不可篡改的日志。
在前端状态页的 **审计日志** Tab 中，你可以随时查看整个项目成长的“设定流水账”。

## 3. 如何安全回滚？

如果你发现设定乱了，或者上一章的 Patch 被错误合并了，请按照以下步骤回滚：
1. 停止在工作台生成新的章节。
2. 点击顶部导航栏进入 **状态 (State) 页**。
3. 点击右侧的 **备份与回滚** Tab。
4. 列表会展示所有的历史备份节点（包含时间、触发原因，例如“Manual edit plot threads”）。
5. 找到你确认完好的那个时间点，点击 **“回滚 (Restore)”**。
6. 系统会将那个时刻的备份完全覆盖现在的 `memory` 设定。

> [!WARNING]
> 回滚操作本身**不可逆**！回滚后，在此时间点之后的所有设定修改将全部丢失。请务必在点击回滚前，确认你选对的时间节点。如果你不确定，可以在回滚前先去顶部点击**导出设定包**在本地存一份。

## 4. 导出设定包

如果你不信任本地存储，或者需要换台电脑工作，你可以点击**导出设定包 (Export Story Bible)**。
系统会将所有干净的、确认过的事实设定组合成一篇 Markdown 格式的故事圣经，可以直接阅读。
注意：导出的设定包是只读归档文件，系统目前不支持从 Markdown 反向导入覆盖设定。
