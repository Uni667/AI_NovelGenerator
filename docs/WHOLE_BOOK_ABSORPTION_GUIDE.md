# 整本吸收工作流指南

## 1. 简介
将一部几十上百万字的长篇小说放入提示词是不可行的，既超出上下文长度也极其昂贵。因此，我们构建了 **整本吸收工作流** (Whole-Book Absorption Pipeline)。它是一组后台常驻的自动化任务，旨在对已扫描的书籍进行逐章提炼、拆解，并压缩为“精华结构”。

## 2. 启动吸收任务
在【本地参考书库】页面，选中一本状态为 `scanned` 或 `partial` 的书，点击“开始提取/吸收精华”。
由于整本小说的分析需要调用大量的 LLM 次数，该过程可能需要 15 分钟到几个小时不等，具体取决于原著篇幅。

## 3. 任务状态与控制
吸收任务具备极高的鲁棒性：
* **后台异步**：你可以关闭网页，任务依然会在后端排队执行。
* **暂停与恢复**：点击任务卡片上的“暂停”按钮可安全中止循环；后续点击“恢复”将从上一个成功的章节断点继续，不会重复消耗 Token。
* **失败重试**：由于模型质量或网络抖动导致的章节解析失败会被标记为 `failed`，但整体任务不受影响。待全书跑完后，系统会自动针对失败章节发起最多2次补偿尝试。
* **取消任务**：若不想继续，可硬性取消，已经解析完毕的章节精华将永久保留。

## 4. 吸收分析层级
整个吸收过程会产出如下层级的分析：
1. **章节摘要**：每章 300-500 字的故事提要。
2. **场景拆解**：该章包含了多少个场景，场景的目的与动作序列。
3. **卷级串联**：合并同一卷（如1-50章）的摘要，并提炼人物弧光和核心冲突。
4. **全书总结**：生成核心世界观、主线脉络。
5. **风格圣经 (Style Bible)**：提取作者行文的描写习惯、常用词汇、对话特色。
6. **场景模板 (Scene Patterns)**：提取经典的高潮反转公式、升级套路等。

这些产物都将被存入指定的精华文件夹中，在未来的新书创作时，作为极致浓缩的 RAG 知识注入给大模型。
