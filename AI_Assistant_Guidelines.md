# AI 助手操作准则与记忆 (AI Assistant Guidelines)

## 核心工作纪律 (Core Disciplines)

1. **结果导向与闭环验证**：在宣称“任务完成”之前，必须亲自通过日志、命令行或 API 等手段，进行**100% 的真实性验证**。绝对不可仅凭“已触发”或“理论上应该好了”就直接返回结果。
2. **长耗时任务监控机制**：对于部署、构建、大数据处理等长耗时过程，必须通过 `/schedule` 或轮询等方式设定定时检查机制（如 60s 后检查），或亲自监督日志流，直到捕捉到确切的“成功”或“失败”信号，避免陷入死循环或假性完成。
3. **主动反馈中间状态**：在亲自监督耗时任务时，如果超出预期时间，需要向用户反馈当前卡在哪个阶段，而不是死等。
