### 核心业务流程（4 阶段生成管线）

```
创建项目 → 设定参数(主题/类型/章节数/字数/平台)
    │
    ├─ [阶段1] 架构生成 (architecture.py)
    │   ├─ 核心种子 (雪花写作法第1层)
    │   ├─ 角色架构 (角色弧光模型, 3-6个角色)
    │   ├─ 世界观构建 (三维度交织法)
    │   ├─ 三幕式情节架构
    │   ├─ 初始全局摘要 / 伏笔暗线台账 / 角色状态表
    │   └─ 每步完成后自动"去策划腔重写"
    │
    ├─ [阶段2] 章节目录生成 (blueprint.py)
    │   ├─ 支持单次或分块生成 (1000+ 章)
    │   └─ 每章含: 定位/作用/悬念密度/伏笔操作/简述
    │
    ├─ [阶段3] 章节草稿生成 (chapter.py) ← 最复杂
    │   ├─ 构建提示词 (读取架构/目录/全局摘要/角色状态/伏笔台账)
    │   ├─ 历史章节摘要 (前3章 → LLM 压缩)
    │   ├─ 知识库语义检索 (关键词提取 → 向量搜索 → 三级过滤)
    │   ├─ LLM 生成草稿
    │   ├─ 自动去 AI 味修订
    │   ├─ 平台质检四维分析 (钩子/中段质量/对话同质化)
    │   └─ 不达标则自动返修
    │
    └─ [阶段4] 定稿 (finalization.py)
        ├─ 更新全局摘要
        ├─ 更新角色状态表
        ├─ 更新伏笔暗线台账
        └─ 写入向量库 (用于后续章节检索)
```

## 三、我的 API 使用场景

### 当前 API 配置方案

我使用 **DeepSeek V4 系列 API** 作为主要 LLM 提供商，通过本地代理路由接入 Claude Code：

```
Claude Code CLI → http://127.0.0.1:8787 (本地路由代理) → api.deepseek.com/anthropic
```

**路由配置文件位置**:
- 路由脚本: `C:\Users\Lenovo\.claude\deepseek-router\router.js`
- 启动脚本: `C:\Users\Lenovo\.claude\deepseek-router\start-router.cmd`
- Claude 启动包装: `C:\Users\Lenovo\.claude\deepseek-router\claude-auto.cmd`
- Claude 设置: `C:\Users\Lenovo\.claude\settings.json`

**环境变量**:
- `DEEPSEEK_API_KEY`: 我的 DeepSeek API Key（用户级环境变量）
- `ANTHROPIC_BASE_URL`: `http://127.0.0.1:8787`（Claude Code 的 API 地址）
- `ANTHROPIC_API_KEY`: `proxy-routed`（占位符，实际由路由转发）
- `ANTHROPIC_MODEL`: `deepseek-v4-auto`（自动选择 flash/pro）

**智能路由策略**:
- 简单任务 → `deepseek-v4-flash`（快速便宜）
- 复杂任务（大上下文 >60K 字符、深度思考、架构/重构/调试、长篇写作）→ `deepseek-v4-pro`
- effortLevel 设置为 `max`（最高思考级别）

### API 用途分类

1. **编码辅助**（通过 Claude Code CLI）:
   - 写新功能、修 bug、重构代码
   - 代码审查、测试编写
   - 项目架构讨论和技术选型

2. **小说生成**（通过项目自身的后端服务）:
   - 架构生成（雪花写作法、世界观构建）
   - 章节目录规划（100-1000+ 章）
   - 章节正文生成（每章数千到数万字）
   - 去 AI 味修订、质量返修
   - 这些调用走项目后端的 `model_runtime.py` → `llm_adapters.py`

3. **Embedding 向量化**:
   - 章节内容向量化存入 SQLite
   - 知识库语义检索
   - 用于长程一致性维护

### 成本敏感度

- **高敏感度**：小说生成场景 token 消耗极大（单次会话可达 162K input tokens, 25M cache read tokens）
- **需要缓存优化**：前缀缓存命中率直接影响成本
- **之前实测**：一次完整会话约 $15.47（使用 deepseek-v4-pro）

## 六、Claude Code API 配置操作手册

> ⚡ **这是我最常需要你帮我做的事情之一**：我给你一个新的 DeepSeek API Key，你需要按以下步骤完成配置。

### 操作流程（当我给你新 API Key 时）

假设我给你的新 Key 是 `sk-xxxxxxxx`，你需要依次执行：

#### Step 1: 写入环境变量

```powershell
# PowerShell 命令，将新 Key 写入用户级环境变量
[Environment]::SetEnvironmentVariable('DEEPSEEK_API_KEY', 'sk-xxxxxxxx', 'User')
```

验证是否写入成功：
```powershell
[Environment]::GetEnvironmentVariable('DEEPSEEK_API_KEY', 'User')
# 期望输出: sk-xxxxxxxx
```

#### Step 2: 停止旧路由进程

```powershell
# 查找并停止占用 8787 端口的旧路由进程
$proc = Get-NetTCPConnection -LocalAddress 127.0.0.1 -LocalPort 8787 -State Listen -ErrorAction SilentlyContinue | Select-Object -First 1
if ($proc) { Stop-Process -Id $proc.OwningProcess -Force; "stopped PID: $($proc.OwningProcess)" } else { "port free" }
```

#### Step 3: 用新 Key 启动路由

```powershell
# 必须先设置环境变量再启动 node，否则路由读不到新 Key
$env:DEEPSEEK_API_KEY = [Environment]::GetEnvironmentVariable('DEEPSEEK_API_KEY', 'User')
node "C:\Users\Lenovo\.claude\deepseek-router\router.js"
```

期望看到输出：
```
[deepseek-router] listening on http://127.0.0.1:8787
[deepseek-router] auto routes to deepseek-v4-flash or deepseek-v4-pro
```

#### Step 4: 验证端到端连通性

```powershell
# 测试通过本地路由转发到 DeepSeek API 是否正常
try {
  $response = Invoke-RestMethod -Uri 'http://127.0.0.1:8787/v1/messages' `
    -Method Post `
    -Headers @{ 'x-api-key' = 'proxy-routed'; 'Content-Type' = 'application/json' } `
    -Body '{"model":"deepseek-v4-auto","max_tokens":10,"messages":[{"role":"user","content":"hi"}]}' `
    -TimeoutSec 15
  "OK - model: $($response.model)"
} catch { "ERROR: $($_.Exception.Message)" }
```

期望输出：`OK - model: deepseek-v4-flash`（或 deepseek-v4-pro）

#### Step 5: 验证健康检查

```powershell
Invoke-RestMethod -Uri 'http://127.0.0.1:8787/health' -TimeoutSec 5
```

期望输出包含 `ok: True` 和三个模型名。

### 完整一键脚本（合并版）

如果需要一步到位，可以用这个合并脚本：

```powershell
# ====== Claude Code DeepSeek API 一键配置 ======
# 用法: 将下面的 sk-xxxxxxxx 替换为实际 Key 后执行

$NEW_KEY = "sk-xxxxxxxx"

# 1. 写入环境变量
[Environment]::SetEnvironmentVariable('DEEPSEEK_API_KEY', $NEW_KEY, 'User')
Write-Host "[1/4] 环境变量已写入: $($NEW_KEY.Substring(0,10))..."

# 2. 停止旧路由
$proc = Get-NetTCPConnection -LocalAddress 127.0.0.1 -LocalPort 8787 -State Listen -ErrorAction SilentlyContinue | Select-Object -First 1
if ($proc) { Stop-Process -Id $proc.OwningProcess -Force; Write-Host "[2/4] 已停止旧路由 PID: $($proc.OwningProcess)" } else { Write-Host "[2/4] 端口 8787 未被占用" }

# 3. 启动新路由（后台运行）
Start-Sleep -Milliseconds 500
$env:DEEPSEEK_API_KEY = [Environment]::GetEnvironmentVariable('DEEPSEEK_API_KEY', 'User')
Start-Process -NoNewWindow -FilePath "node" -ArgumentList "`"C:\Users\Lenovo\.claude\deepseek-router\router.js`""
Write-Host "[3/4] 新路由已启动..."

# 4. 等待启动后验证
Start-Sleep -Seconds 2
try {
  $r = Invoke-RestMethod -Uri 'http://127.0.0.1:8787/v1/messages' -Method Post `
    -Headers @{ 'x-api-key' = 'proxy-routed'; 'Content-Type' = 'application/json' } `
    -Body '{"model":"deepseek-v4-auto","max_tokens":10,"messages":[{"role":"user","content":"hi"}]}' -TimeoutSec 15
  Write-Host "[4/4] 验证通过! 模型: $($r.model)" -ForegroundColor Green
} catch {
  Write-Host "[4/4] 验证失败: $($_.Exception.Message)" -ForegroundColor Red
}
# ====== 结束 ======
```

### 常见问题排查

| 症状 | 原因 | 解决方法 |
|------|------|---------|
| `ConnectionRefused` | 路由没启动或已崩溃 | 执行 Step 2 + Step 3 重启 |
| `401 Unauthorized` | 路由启动时没读到新 Key（用的旧 Key 或空 Key）| 必须确保 `$env:DEEPSEEK_API_KEY` 在 `node router.js` **之前**设置 |
| `EADDRINUSE: port 8787` | 旧进程没杀干净 | 先执行 Step 2 强制停止 |
| `listen EADDRINUSE` | 同上 | 同上 |
| 路由正常但 Claude Code 连不上 | Claude Code 的 settings.json 中 ANTHROPIC_BASE_URL 不是 `http://127.0.0.1:8787` | 检查 `C:\Users\Lenovo\.claude\settings.json` |
